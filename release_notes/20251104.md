# Release Notes - November 4, 2025

## Summary

This release introduces **automatic OS detection with intelligent mouse wheel direction adjustment**, horizontal mouse scroll support, and enhances screen capture workflow efficiency through strategic mouse button placement. The update fixes critical build errors by migrating to RGB Matrix keycodes and ensures full QMK standards compliance for mouse controls.

---

## Key Changes

### 1. **Automatic OS Detection & Mouse Wheel Direction Adjustment** ğŸ†•

**Problem Solved**: Windows and macOS have opposite mouse wheel scrolling directions. Using a single firmware across both operating systems resulted in counter-intuitive scrolling behavior - scrolling down on one OS would scroll up on the other.

**Solution**: Implemented QMK's OS detection feature with automatic mouse wheel direction inversion for macOS/iOS.

**Technical Implementation**:
- **OS Detection**: Automatically detects operating system after USB connection (hundreds of milliseconds)
- **Automatic Adjustment**: Inverts vertical and horizontal scroll directions for macOS/iOS to match their "natural scrolling" behavior
- **Visual Feedback**: RGB Matrix displays color-coded OS identification
  - **White** (HSV: 0, 0, 255) - macOS/iOS detected
  - **Blue** (HSV: 170, 255, 255) - Windows detected
  - **Green** (HSV: 85, 255, 255) - Linux detected
  - **Yellow** (HSV: 43, 255, 255) - Unknown/Unsure OS

**Code Implementation**:
```c
// OS detection callback with RGB visual feedback
bool process_detected_host_os_user(os_variant_t detected_os) {
    current_os = detected_os;
    os_detected = true;

    #ifdef RGB_MATRIX_ENABLE
    switch (detected_os) {
        case OS_MACOS:
        case OS_IOS:
            rgb_matrix_sethsv_noeeprom(0, 0, 255);  // White
            break;
        case OS_WINDOWS:
            rgb_matrix_sethsv_noeeprom(170, 255, 255);  // Blue
            break;
        // ... (additional cases)
    }
    #endif
    return true;
}

// Automatic wheel direction adjustment
report_mouse_t pointing_device_task_user(report_mouse_t mouse_report) {
    if (os_detected && (current_os == OS_MACOS || current_os == OS_IOS)) {
        mouse_report.v = -mouse_report.v;  // Invert vertical scroll
        mouse_report.h = -mouse_report.h;  // Invert horizontal scroll
    }
    return mouse_report;
}
```

**Build Configuration**:
```makefile
# rules.mk
OS_DETECTION_ENABLE = yes
```

**Benefits**:
- **Cross-Platform Consistency**: Natural scrolling behavior on both Windows and macOS
- **Zero Configuration**: Automatic detection and adjustment - no user intervention needed
- **Visual Confirmation**: RGB color indicates detected OS for troubleshooting
- **Seamless Experience**: Same firmware works naturally across different operating systems

### 3. **Enhanced Screen Capture Workflow**

**New Feature**: Added mouse buttons at Fn3+Shift position for one-handed screen capture area selection.

**Strategic Button Placement**:
- **Row 3, Column 1**: `MS_BTN2` (Right-click) at Left Shift position
